FCM Web: legacy API (getToken / deleteToken) and FID API (register / unregister)

Based on quickstart-js/messaging: the page script uses the workspace firebase package. public/firebase-messaging-sw.js loads the matching gstatic ESM build (12.13.x) so the service worker can register with a root-compatible scope. Before first use, from the repo root run yarn and yarn build --scope firebase --include-dependencies (or full yarn build) to build packages/firebase dist.
To deploy on Firebase Hosting, use firebase.json in this folder: run yarn firebase:install-cli once, then yarn firebase:login, ensure the monorepo firebase package is built from the repo root, then yarn deploy (Hosting must be enabled for fcm-js-sdk-test in .firebaserc).

A default Web Push (VAPID) public key is set in src/config.ts. To use a different key, set VITE_VAPID_KEY in .env.local and restart yarn dev.
For yarn firebase:login / yarn deploy, run yarn firebase:install-cli once: it installs the Firebase CLI under cli-runner/ with npm (isolated from the monorepo root’s hoisted firebase-tools@13, which crashes on Node 25). If the CLI still fails, use Node 22 (nvm use in this folder).
If deploy returns 401, run yarn firebase:login:reauth (or yarn firebase:logout then yarn firebase:login) and ensure your Google account has access to the project in Firebase Console → Project settings → Users and permissions.
